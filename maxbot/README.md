# Объединённый патч v3+v4

Применяется на проект v2 (после которого нужны контролёры + редактирование).
Содержит все изменения из патчей v3 и v4 сразу.

## Что внутри

### Новое в коде
- **Роль «контролёр»** — отдельная учётка с доступом только к QR-сканеру назначенных мероприятий
- **Управление учётками** (`/admin/users`) — создание, редактирование ФИО/пароля, отключение/активация
- **Загрузка контролёров XLSX** на карточке мероприятия (только для организатора)
- **QR первым сообщением в боте** + меню вторым
- **Новые кнопки «Мои записи»**: QR, отмена с названием мероприятия, переключение уведомлений

### Что появляется в БД
- Таблица `controllers` (логины контролёров)
- Таблица `event_controllers` (привязки N:M к мероприятиям)
- Расширен ENUM `audit_action` на 5 новых значений

## Как накатить

### Шаг 1. Распакуй ZIP поверх проекта

В PowerShell:

```powershell
cd "C:\Users\user\Desktop\maxbot (1)\maxbot"
```

Распакуй `maxbot_patch_combined.zip` сюда **с заменой** существующих файлов.
Windows спросит про замену — отвечай «да для всех».

### Шаг 2. Применяй миграцию БД

```powershell
.venv\Scripts\Activate.ps1
python -m migrations.add_controllers
```

Должно вывести:
```
Применяем миграцию: controllers + event_controllers + новые enum-значения
ALTER TYPE audit_action ADD VALUE ... (5 раз)
CREATE TABLE controllers
CREATE TABLE event_controllers
CREATE INDEX ix_event_controllers_controller
Миграция применена.
```

Миграция **идемпотентна** — можно запускать повторно без вреда.

### Шаг 3. Перезапусти сервер

В окне 1 (сервер): **Ctrl+C** → `python -m app.main`

Ngrok, webhook MAX, PostgreSQL — НЕ перезапускать.

## Что проверить после накатки

1. Войди как **админ** → в шапке появилась ссылка «Пользователи»
2. `/admin/users` → форма создания + три таблицы (организаторы, контролёры, техадмины)
3. У каждой строки: кнопки «✏️ Изменить» и «🔒 Отключить»
4. У своей же учётки кнопки «Отключить» нет — `(это вы)`
5. Создай контролёра через форму (роль «Контролёр»)
6. Войди как **организатор** → открой мероприятие → секция «Контролёры мероприятия» с загрузкой XLSX
7. Войди как **админ** → открой мероприятие → секция «Контролёры мероприятия» **НЕ должна быть видна**
8. В боте → запись → должен прийти QR-картинка первым + меню вторым
9. «Мои записи» в боте → у каждой кнопки `🎫 QR`, `❌ Отменить «Название»`, `🔕/🔔`

## Формат XLSX для загрузки контролёров

| email | full_name |
|---|---|
| controller1@mirea.ru | Иванов Иван |
| controller2@mirea.ru | Петров Пётр |

Шапка (первая строка) пропускается. Email должны **уже существовать** в системе (созданы техадмином заранее).
